Introduction

Conçu comme un bon point de départ, Baselayer peut être tout ce dont vous avez besoin pour les petits projets tels que les sites Web statiques et les blogs. Ou vous pouvez l'utiliser comme baselayer pour démarrer rapidement votre méga projet.

Prêt à être utilisé tel quel, la feuille de style 24 Ko (minifiée) de Baselayer vous donne tout cela :

  • Une réinitialisation CSS moderne
  • Typographie sans classe de style minimaliste et éléments de forme
  • Fonctionnalités d'accessibilité fiables pour les utilisateurs de technologies d'assistance
  • Un système léger de classes d'utilité pour contrôler les dimensions, le positionnement, l'espacement, les bordures, le texte et les images
  • Requêtes @container query utilitaires de mise en page réactive à l'aide de la grille CSS et de la boîte flexible
  • Typographie, mise en page et espacement réactifs à l'aide de fonctions clamp()
  • Un système compact de luminosité des couleurs basé sur color-mix(in OKLCH) pour le texte, les bordures et les arrière-plans
  • Mode sombre intégré utilisant light-dark()
  • Contrôle de thématisation à l'aide de variables CSS
  • Et plus…

Technologies CSS modernes

Supports de la couche de base (baseline) uniquement à jour (2023 en avant) compatibilité de base pour les navigateurs (Safari, Firefox, Chrome, Edge, etc.). Such as:

  • Container queries (inline-size)
  • CSS Grid
  • Math functions (e.g. clamp)
  • Custom properties (variables)
  • color-mix() function
  • light-dark() function
  • CSS class nesting
  • Cascade layers

Baselayer’s design philosophy

La philosophie de conception derrière le projet Baselayer CSS est la suivante :

  • Visez à être un bon point de départ - un baselayer pour un projet de conception Web.
  • Étant si petit (24 Ko), il y a moins que vous devez surcharger afin de le styliser à votre façon (et la plupart des choses dans Baselayer peuvent être surchargés par des variables CSS). Ainsi, il n'y a pas besoin de pré-processurs pour supprimer une lourde charge utile de styles Baselayer inutilisés.
  • PostCSS only — no preprocessors.

Construit en utilisant PostCSS

Baselayer est construit en utilisant PostCSS et quelques plugins. Les plugins utilisés par Baselayer sont :

  • postcss-import — afin que Baselayer puisse être construit à partir de fichiers CSS séparés
  • cssnano — pour supprimer les commentaires et réduire la taille du fichier baselayer.min.css

Note: ni postcss-preset-env ni autoprefixer ont été utilisés dans Baselayer. Et les préprocesseurs tels que Sass, Less, Stylus, etc. ne sont pas nécessaires.

Une réinitialisation CSS moderne

La réinitialisation de Baselayer est une combinaison du « meilleur des deux » de Custom CSS reset de Josh W Comeau et d’Andy Bell (more) modern CSS reset.

Sur cette base, Baselayer s'occupe ensuite des bases en définissant une typographie et des styles minimalistes et faciles à lire pour les boutons et les formulaires.

Baselayer’s built-in accessibility features

Baselayer has two “must have” accessibility features built-in.

(1.) Focus states

After experimenting and testing various focus state styles, I have decided to base Baselayer’s on the ones in the UK National Health Service NHS.uk design system and the UK Government GOV.UK design system.

  1. Links receive an amber background, black text and a thick black underline on :focus-visible — so that the element is clearly visible in a wide range of contexts.

  2. Interacive elements such as form inputs, form buttons, collapsible details, and elements with overflowing content have what looks like a yellow and black “double border” that appears on :focus (actually its created by an outline and a box shadow).

Note: A <button> that is not within a <form>, and a <button type="button"> will not receive the :focus ring (but it will receive the ring on :focus-visible). Ths is true for most browsers except Safari: Safari does not show :focus rings on buttons, and it does not place tabbing focus on overflowing elements, or on the color-picker input.

These focus states are visible on a wide range of colored backgrounds, in both light and dark themes. (See also colors and accessibility.) A z-index raise has been added to prevent the focus ring tucking under a neighbouring item, e.g. in input/button groups.

(2.) The visually-hidden class

The visually-hidden is used to provide additional content for screen-readers, for improved accessibility. For example, it is best practice to have “skip link” above the top navigation bar on your website, but have it (visually) hidden for ably sighted users.

<a href="#main-content"
  tabindex="1"
  class="block p-2 visually-hidden"
>
  Skip to page content
</a>

<!-- Logo and site navigation menu here -->

<div id="main-content">
  ...
</div>
